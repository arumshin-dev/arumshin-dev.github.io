GitHub Pages는 무료 호스팅, 높은 커스터마이징 가능성, 코드 공유에 유리한 절충안으로 소개됨.

## GitHub Pages 블로그 생성 과정

1. GitHub 계정 생성
2. 마음에 드는 테마를 선택합니다. https://github.com/topics/jekyll-theme
3. fork 합니다.
도큐먼트를 꼼꼼히 확인합니다.
인기 테마(예: Minimal Mistakes)를 포크하여 복사

4. 저장소 이름을 아이디.github.io 형식으로 변경
5. _config.yml 파일에서 url을 수정합니다.
_config.yml 파일 수정 → 블로그 기본 설정, 광고, SNS 연동 가능
6. 새로운 포스팅을 생성합니다.
https://jekyllrb.com/docs/posts/
`----
layout: single
title: 을 적습니다.
----`
_posts 폴더에 마크다운 파일 작성 → 포스팅 발행

## 포스팅 작성 및 관리

- 마크다운 문법을 활용해 글 작성(https://teddylee777.github.io/jekyll/Jekyll-%EC%82%AC%EC%9A%A9%EC%9D%84-%EC%9C%84%ED%95%9C-markdown-%EB%AC%B8%EB%B2%95/)

- Jupyter Notebook 결과물을 마크다운으로 변환해 쉽게 업로드 가능. jupyter notebook 파일을 .md로 다운로드 받아 업로드 합니다.

- 이미지 업로드 및 테마 변경도 설정 파일에서 간단히 가능 _config.yml 에 테마 있음

## 추가 도구 활용

- Typora 같은 마크다운 에디터를 사용하면 문법을 몰라도 쉽게 글 작성 가능

- 광고(AdSense) 연동, SNS 계정 연결, 구글 애널리틱스 설정 등 확장 기능 제공

## 장점

- 무료 호스팅

- 코드와 데이터 시각화 결과를 그대로 블로그에 반영 가능

- 빠른 구축(10분 내 완성)과 높은 확장성
